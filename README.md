## 📌 서비스 소개

온라인에서 다양한 플랫폼(네이버 스토어, 인스타그램 등)을 운영하는 **개인 자영업자**를 대상으로,
각 플랫폼별 성과를 분석하고, 고객 반응에 기반한 **마케팅 문구**를 **자동 추천** 및 **카카오톡으로 발송**할 수 있는 플랫폼을 구축한다.

## 프로젝트 기획 방향
<details>
<summary><b> 프로젝트 목적</b></summary>
<br/>
온라인에서 다양한 플랫폼(네이버 스토어, 인스타그램 등)을 운영하는 개인 자영업자를 대상으로, 각 플랫폼별 성과를 분석하고, 고객 반응에 기반한 마케팅 문구를 자동 추천 및 카카오톡으로 발송할 수 있는 플랫폼을 구축한다.
<br/>
플랫폼 가입시 사용자 자신의 채널(스토어) 정보를 등록하며, 이에 따라 홍보 메시지 추천, 발송, 반응 분석까지 자동화한다.
<br/>
추가 정보 등록은 개인 정보 수정에서 추가할 수 있다.
</details>

## 소감 

조선영 : 기본적인 테이블 구성과 Git으로 협업을 시작했지만, 이번 프로젝트를 통해 ERD, 컬럼 설명, 제약조건 등 세부 문서화의 정확성과 일관성이 협업에서 얼마나 중요한지 다시 한 번 느꼈습니다. 
추후에는 임시저장, 게시글, 동영상 테이블까지 확장하여 구조를 더욱 구체화하고자 합니다.

유지선 : 이번 기획 프로젝트에서는 피그마(Figma)를 활용한 화면 UI 제작과 서비스 플로우 설계를 중점적으로 담당했다. 
이번 경험을 통해 기획의 전반적인 흐름과 구조를 이해할 수 있었던 점에서 의미있는 경험이었다.
다만, 아직 피그마를 완벽하게 숙련하지 못한 상태에서 UI를 기획해야 했기 때문에 작업에 어려움을 느끼기도 했다.
또한, 전반적인 일정이 촉박해 예상했던 기대치만큼 결과물을 완성하지 못한 점이 아쉬웠다.

이찬웅 : 이번 프로젝트를 통해 Flask 기반의 백엔드 서버를 구축하고, 다양한 외부 API를 연동하는 개발 과정을 경험한 점이 인상적이었습니다. 
또한, 데이터를 시각화하며 마케팅 데이터에 대한 분석자료를 얻는 과정은 실질적인 분석 역량까지 넓힐 수 있는 좋은 기회였습니다.

윤상현 : 이번 팀 프로젝트를 진행하면서 아쉬움이 남았습니다.
처음 해 보는 IT계열 프로젝트여서 그런지 생소하기도 하고 어떻게 방향을 잡아야 하는지, 어떤 진행 과정을 거쳐야 하는지 제대로 파악하지 못한 것 같습니다.
또한 스스로 기준치와 같은 목표를 너무 낮게 설정해서 더 깊게 생각하지 못했던 점도 아쉽습니다. 조금 더 적극적으로 기획에 참여하고 고심해 보며, 좀 더 세부적인 요소에서 구현을 어떻게 진행할지 고민했다면 어땠을까 싶지만 
그럼에도 이러한 경험을 통해 부족한 부분이 무엇인지 확인 할 수 있어서 좋은 경험이었습니다.
